Ver Especificación Completa

1. Introducción

📋 1.1 Propósito

EcoSwap es un sistema web y móvil diseñado para incentivar el reciclaje mediante un sistema de recompensas. Los usuarios pueden escanear y depositar residuos reciclables a cambio de puntos, que pueden canjear por descuentos en negocios locales.

🎯 1.2 Alcance

El sistema permitirá a los usuarios:

  • Escanear residuos reciclables con visión artificial
  • Depositar residuos en puntos de reciclaje asociados
  • Acumular puntos por cada material reciclado
  • Canjear puntos por descuentos o beneficios en comercios aliados
  • Consultar su historial de reciclaje e impacto ambiental

📚 1.3 Definiciones, Acrónimos y Abreviaturas

Término Definición
IA (Inteligencia Artificial) Algoritmos de aprendizaje automático para reconocimiento de residuos
API (Interfaz de Programación de Aplicaciones) Conjunto de funciones para interactuar con otros servicios
Usuario Persona que utiliza la aplicación para reciclar y obtener beneficios
Negocios Aliados Empresas que ofrecen beneficios a cambio de reciclaje

2. Descripción General

🌐 2.1 Perspectiva del Producto

EcoSwap es una plataforma SaaS (Software como Servicio) que conecta usuarios con puntos de reciclaje y comercios interesados en promover la economía circular.

Características Principales:

Plataforma integral que combina tecnología de IA, gamificación y sostenibilidad para crear un ecosistema de reciclaje inteligente.

⚡ 2.2 Funcionalidades del Producto

👥

Registro de Usuarios

Registro e inicio de sesión de usuarios y negocios aliados

🤖

Escaneo IA

Escaneo de residuos mediante inteligencia artificial

🎯

Sistema de Puntos

Acumulación de puntos por reciclaje

🎁

Canje de Recompensas

Canje de puntos en negocios participantes

📊

Historial

Historial de reciclaje e impacto ambiental

🗺

Mapas

Integración con mapas para localizar puntos de reciclaje

👤 2.3 Características de los Usuarios

👥 Usuarios Regulares

Personas que reciclan y obtienen beneficios por sus acciones sostenibles.

  • Edad: 16-65 años
  • Nivel tecnológico: Básico a intermedio
  • Motivación: Beneficios económicos y conciencia ambiental

🏪 Negocios Aliados

Comercios que ofrecen recompensas y promociones especiales.

  • Tipo: Comercios locales y cadenas
  • Objetivo: Atraer clientes conscientes del medio ambiente
  • Beneficio: Marketing verde y responsabilidad social

👨‍💼 Administradores

Gestionan la plataforma y validan nuevos puntos de reciclaje.

  • Rol: Supervisión y mantenimiento del sistema
  • Responsabilidades: Validación y moderación
  • Acceso: Panel de administración completo

⚠ 2.4 Restricciones

  • El reconocimiento de residuos debe realizarse en dispositivos con cámara funcional
  • La acumulación de puntos está sujeta a validación automática y manual
  • Requiere conexión a internet para sincronizar datos en la nube
  • Los puntos tienen fecha de expiración para mantener la actividad del usuario

🔗 2.5 Suposiciones y Dependencias

  • Se integrará con Google Maps para la localización de puntos de reciclaje
  • Se usará TensorFlow Lite para el reconocimiento de imágenes
  • Los usuarios tendrán acceso a dispositivos con cámara y conexión a internet
  • Los negocios aliados mantendrán sus ofertas actualizadas

3. Requisitos Específicos

⚙ 3.1 Requisitos Funcionales

RF1: El sistema debe permitir el registro y autenticación de usuarios mediante correo electrónico y contraseña.
RF2: El sistema debe permitir a los usuarios escanear residuos mediante la cámara del dispositivo y visión artificial (IA).
RF3: El sistema debe clasificar automáticamente los residuos escaneados por tipo (plástico, vidrio, papel, metal, etc.).
RF4: El sistema debe asignar puntos automáticamente según el tipo y cantidad del residuo reciclado.
RF5: El usuario debe poder consultar su saldo actual de puntos acumulados en tiempo real.
RF6: El sistema debe permitir a los usuarios canjear puntos por beneficios en los negocios aliados.
RF7: El sistema debe mostrar un historial detallado del reciclaje del usuario, incluyendo fechas, tipos de residuos y puntos ganados.
RF8: El sistema debe permitir a los administradores validar nuevos puntos de reciclaje y usuarios especiales (como negocios aliados).
RF9: El sistema debe integrar un mapa interactivo con la ubicación de los puntos de reciclaje más cercanos.
RF10: El sistema debe enviar notificaciones al usuario sobre promociones o beneficios disponibles en comercios aliados.

🚀 3.2 Requisitos No Funcionales

RNF1: El tiempo de respuesta del sistema no debe superar los 2 segundos en dispositivos móviles estándar.
RNF2: La disponibilidad del sistema debe ser del 99.5% mensual como mínimo.
RNF3: La interfaz de usuario debe ser accesible para personas con discapacidades visuales o motrices.
RNF4: Los datos del usuario deben almacenarse cifrados y cumplir con la normativa de protección de datos (por ejemplo, GDPR).
RNF5: El sistema debe ser compatible con Android 8 o superior y iOS 12 o superior.
RNF6: La aplicación debe funcionar en modo responsivo para ajustarse correctamente en móviles, tabletas y computadoras.
RNF7: La interfaz debe tener un diseño amigable con colores ecológicos y botones grandes para facilitar la navegación.
RNF8: Las actualizaciones del sistema no deben interrumpir el servicio durante más de 5 minutos.
RNF9: El sistema debe utilizar servicios en la nube (AWS o Google Cloud) para garantizar escalabilidad.
RNF10: El sistema debe integrarse correctamente con APIs externas como Google Maps y TensorFlow Lite sin errores críticos.

🎨 3.3 Requisitos de Interfaz de Usuario

Log de Eventos del Proyecto

📊 Log de Eventos del Proyecto (Enfoque en Status Code)

Esta tabla está diseñada para registrar eventos significativos, movimientos y posibles problemas dentro del proyecto EcoSwap, prestando especial atención a los códigos de estado de las operaciones.

Nombre de la Tabla Sugerido: log_eventos_proyecto

Esta tabla centraliza el registro de todas las operaciones críticas del sistema, facilitando el monitoreo, diagnóstico y auditoría de eventos.

Columna Tipo de Dato Descripción Ejemplo
id_log UUID o BIGINT (Autoincremental) Identificador único para el registro de log (Clave Primaria). a1b2c3d4-e5f6-7890-1234-567890abcdef o 2001
fecha_hora_evento TIMESTAMP WITH TIME ZONE Fecha y hora exactas (con zona horaria) en que ocurrió el evento. 2025-05-28 15:00:00-0500
componente_afectado VARCHAR(100) Módulo, funcionalidad, entidad o recurso específico del proyecto afectado por el evento. 'API_Usuarios', 'Modulo_Pagos', 'Registro_Cliente_ID123'
tipo_evento VARCHAR(100) Describe la acción o tipo de evento ocurrido (ej. 'CREACION_USUARIO', 'ACTUALIZACION_PERFIL', 'PAGO_RECHAZADO', 'ACCESO_DENEGADO', 'ERROR_PROCESO_BATCH', 'TAREA_PROGRAMADA_FIN'). 'ACTUALIZACION_PERFIL'
id_usuario_o_sistema VARCHAR(50) Identificador del usuario que realizó la acción, o un identificador para procesos automáticos del sistema (ej. 'SISTEMA', 'CRON_JOB_X'). 'juan.perez', 'SISTEMA_CRON_FACTURACION'
status_code_operacion INT Código numérico que representa el estado o resultado de la operación. Puede ser un código HTTP (si aplica), un código de salida de un script, o un código interno definido por el proyecto. 200 (OK HTTP), 401 (No Autorizado HTTP), 503 (Servicio No Disponible), 0 (Éxito Proceso Interno), 3001 (Error Validación Interno)
descripcion_status VARCHAR(255) Descripción textual clara y concisa del status_code_operacion. "El perfil del usuario fue actualizado correctamente.", "Error interno del servidor al procesar la solicitud.", "El proceso de facturación nocturna finalizó sin errores."
mensaje_detallado TEXT Detalles adicionales sobre el evento, incluyendo datos relevantes para el diagnóstico, valores cambiados (si aplica), o mensajes de error específicos (ej. stack trace). "Cambio de email para usuario 'juan.perez' de 'old@example.com' a 'new@example.com'.", "NullPointerException en com.proyecto.ModuloPagos.procesar() linea 42"
ip_origen VARCHAR(45) (Opcional) Dirección IP desde la cual se originó la solicitud o el evento, si es relevante (especialmente para acciones de usuario). '192.168.1.100', '2001:0db8:85a3:0000:0000:8a2e:0370:7334'

Implementación y Uso:

Esta tabla de registro es fundamental para:

  • Monitoreo en tiempo real del estado del sistema
  • Auditoría de acciones de usuarios y procesos
  • Diagnóstico de problemas y errores
  • Análisis de patrones de uso y comportamiento
  • Cumplimiento de requisitos de seguridad y normativas

4. Apéndices

📊 4.1 Diagramas de Flujo de Interacción

Los diagramas de flujo muestran la interacción del usuario con el sistema:

  1. Usuario abre la aplicación y se autentica
  2. Escanea un residuo con la cámara del dispositivo
  3. IA procesa la imagen y clasifica el residuo automáticamente
  4. Sistema asigna puntos según el tipo y cantidad
  5. Usuario confirma el depósito en punto de reciclaje
  6. Puntos se acreditan en la cuenta del usuario
  7. Usuario puede canjear puntos por recompensas

📋 4.2 Casos de Uso Detallados

Caso de Uso Principal: Reciclaje de Residuo

Actor: Usuario registrado

Precondición: Usuario autenticado con cámara funcional

Flujo Principal:

  1. Usuario selecciona "Escanear Residuo"
  2. Sistema activa la cámara
  3. Usuario enfoca el residuo y toma foto
  4. IA analiza y clasifica el residuo
  5. Sistema muestra tipo detectado y puntos a ganar
  6. Usuario confirma y deposita en punto de reciclaje
  7. Sistema acredita puntos en la cuenta

Postcondición: Puntos acreditados y registro en historial
